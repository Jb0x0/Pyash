Pyash only works on Unix based Operating Systems (so Linux, BSD, MacOS, etc).

Please ensure that Python (3.x) and Colorama are installed prior to running Pyash.py

General Use:

    Every time that Pyash.py runs it'll automatically look for a file in your home (~)
    directory called ".pyash_shell_path", if it doesn't find the file it'll boot into
    the Setup Wizard.

    Once in the Setup Wizard, you'll be greeted by a basic shell with commands
    like "cd", "ls", "pwd", "select", etc. To select where to put the "shell"
    either type "select" to choose the directory that you're currently in, or
    "select <dir>" to choose a directory in the directory you're currently in.
    All other commands in the Setup Wizard are self explanatory.

    After completing the Setup Wizard you'll be greeted by the full shell environment.
    Type "help" to view base commands, type "extras" for the base extras and custom
    commands. Most commands are self explanatory, but local manual pages are available
    for all commands via the "man" command (usage: man <cmd>).

    Pyash has support for running ".sh" Bash files, simply be in the same directory
    as the ".sh" file and type: "chmod +x example.sh" then type: "./example.sh".
    You can edit ".sh" files or any other text files with the built in text editor:
    "Pyim". (usage: pyim <file>)

How to add custom commands:

    After completing the Setup Wizard, Pyash automatically creates two hidden files
    in your "shell" folder, ".pyash_history" and ".pyash_extras". To add your own
    custom commands to Pyash, you must edit the ".pyash_extras" file.
    instructions on how to format your commands are noted in ".pyash_extras",
    it needs to be formated as: <command_name> [package_name]

Q&A:

    Q: Will Pyash ever be compatiable with Windows (11, 10, etc)?
    A: No, there are no current plans to ever port Pyash to Windows,
       feel free to fork Pyash and make it Windows compatiable if you want

    Q: Will you ever rewrite Pyash in a different coding language?
    A: No, Python was chosen for a specific reason, for its modularity and extensive
       Standard Libary.

    Q: Will redownloading Pyash from GitHub to update it mess with my current "shell"
       directory?
    A: No, all modern Pyash versions (Versions 1.0 and beyond), will be able to access
       your "shell" directory without causing any issues

    Q: How often will Pyash be updated/improved?
    A: Decently often, expect updates every few weeks or months. Open GitHub issues
       and pull request are reviewed every few weeks as well (sometimes sooner).

    Q: Am I allowed to fork and modify Pyash?
    A: Yes, under the MIT License you are allowed full rights to use, modify, study,
       and distribute the Pyash codebase in both private and commercial environments.
